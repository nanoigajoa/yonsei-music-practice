## Project Overview

음악대학 연습실 보조 서비스. 현재 키오스크 기반 예약 시스템의 대기 문제를 해결하기 위한 웹/앱 서비스.

### 현행 시스템 제약

- 키오스크 3대 (엘리베이터 앞 층마다 배치), 50여 개 방
- 방 예약: 최대 2시간, 연장 1회(최대 2시간), 1일 4시간 한도
- **필수**: 실물 학생증을 방 앞 단말기에 태그해야 개방 (변경 불가)
- 예약 후 10분 이내 태그 필수, 미반납 시 패널티 (2회 → 3일, 3회 → 7주 사용 불가)
- 학교 보안 시스템 연동 불가, 모바일 학생증으로 대체 불가

### 핵심 문제

1. 혼잡 시 키오스크 대기 6~10명 (예약 + 연장 모두)
2. 실물학생증 태그를 위해 왕복 이동 필요
3. 태그 지연 시 방 선점 분쟁
4. 반납 미실시 패널티

### 솔루션 범위 (학교 시스템 건드리지 않는 서비스)

| 기능 | 설명 |
|------|------|
| **양도/매칭** | 방 빌렸지만 사용 안 할 때 다른 사용자와 연결 (식사·공강 등) |
| **바꿔치기 매칭** | 연장 후 카드 교환할 상대 실시간 모집 |
| **조기 반납 예고** | "30분 뒤 나가요" 등 사전 예고 → 대기자가 미리 키오스크 줄 준비 |
| **대기 현황** | 층별 키오스크 대기 인원 크라우드소싱 공유 |
| **방 공실 현황** | 사용자 보고 기반 실시간 공실 추정 (공식 데이터 없음) |
| **방 비품 정보** | 피아노의자·일반의자·보면대 개수를 방별로 표시 |
| **반납 리마인더** | 예약 시간 X분 전 푸시 알림 |

### 방 분류 규칙

- 피아노과 전용 방: 피아노과만 예약 가능
- 교회음악과 전용 방: 교회음악과만 예약 가능
- 나머지 방: 관현악과·성악과·작곡과 모두 예약 가능

## Architecture (계획)

- **Frontend**: 웹 우선 (모바일 반응형), 향후 PWA
- **Backend**: 경량 API 서버 (실시간 기능 필요 → WebSocket 또는 SSE)
- **DB**: 예약 매칭·크라우드소싱 데이터 저장
- **Auth**: 학번 기반 간이 인증 (학교 SSO 연동 불가 전제)
